OBJETIVO

El alumno aprenderá sobre la importancia que tiene la realización de pruebas del software, como un proceso que se incluye dentro el control de calidad. Conocerá las estrategias para planear casos de prueba de software, las técnicas para la aplicación de las pruebas, y la forma adecuada, según el estándar IEEE, para realizar un plan de pruebas y documentar dichas pruebas. Además de conocer los puntos estratégicos que dan paso al análisis de Errores generados.


Introducción

"El software tiene una media de 0,150 errores por cada 1.000 líneas de código"
El control de la calidad es de suma importancia en todo proyecto de software, ya que a partir de ella se pueden verificar y corregir diversas problemáticas que no fueron visualizadas desde el comienzo del proyecto. Para encontrarlas y corregirlas es necesario llevar a cabo un "plan de pruebas": aplicar diversas estrategias para realizarlas. Es a partir de este proceso donde se puede comprobar el correcto funcionamiento del proyecto, es decir, si cumple con todos los requerimientos que se tenían desde el inicio.
