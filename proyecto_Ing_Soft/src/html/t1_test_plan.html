Plan de pruebas

El plan de pruebas y las estrategias de pruebas


Definición: Una prueba se define como un conjunto de actividades previamente planteadas y que se van realizando de manera sistemática, es decir, de forma ordenada.[Pressman 2007]

Plan de pruebas (según él estándar IEEE std. 829 [IEEE 1998])


Se encuentra dentro de la documentación del Diseño de pruebas y se divide en dos secciones:
Objetivo del documento: aquí se señalan el enfoque, los recursos y el esquema de actividades de prueba, así como los elementos a probar, las características, las actividades de prueba, el personal responsable y los riesgos asociados.
La estructura fijada en el estándar:

  • identificador único del documento
  • introducción y resumen de elementos y características a probar
  • elementos de software que se probaran
  • características que se probaran
  • características que no se probaran
  • enfoque general de la prueba
  • criterios de paso/fallo para cada elemento
  • criterios de suspensión y requisitos de reanudación
  • documentos a entregar
  • actividades de preparación y ejecución de pruebas
  • actividades de preparación y ejecución de pruebas
  • responsabilidades en la organización y realización de pruebas
  • necesidades de personal y de formación
  • esquema de tiempos
  • riesgos asumidos por el plan y planes de contingencias para cada riesgo
  • aprobaciones y firmas con nombre y puesto desempeñado
